What this tool does

Gemba-DLCA models demand, load and capacity at each step of your lab pathway. It tells you whether your bottleneck is bench space, staff hours, or a particular grade of staff — and it handles realistic case mixes where Band 6 BMS, Advanced Practitioners, and Consultants handle different case types.

This beta focuses on cut-up and dissection. You can model multiple benches, multiple staff pools, and weekly demand by case type — then see immediately which resource is your binding constraint.

Getting started
  1. 1
    Launch the app using the button above. It runs entirely in your browser — nothing is installed and nothing leaves your device. Read the user guide first if you want a walkthrough.
  2. 2
    Load a demo scenario using the Import button in the top bar. You have been sent two JSON files — start with dlca-demo-bench-binds.json to see a scenario where the benches are the binding constraint, then try dlca-demo-staff-binds-skillmix.json to see skill-mix routing in action.
  3. 3
    Go to Tab 3 — Calculations and click Calculate. The results table shows demand, capacity, and the binding constraint for each step and period.
  4. 4
    Adjust and explore — change staff numbers, bench sessions, or case volumes and recalculate. The tool updates instantly.
  5. 5
    Export your work using the Export button. This saves everything as a JSON file you can reload later or share with a colleague.
