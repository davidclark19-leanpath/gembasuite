Map your process first. Every swim lane is a resource type or functional area. Steps placed in swim lanes identify who does what — and where rework loops generate hidden demand.
A
Swim Lanes (Resource Areas)
Define the functional areas or staff types — each becomes a horizontal row on your process map.
Process Map — Swimlane View
B
Work Unit Tiers
Define up to three hierarchical work units. The primary tier is always active. Enable secondary and tertiary tiers to model multipliers across the pathway (e.g. cases → blocks → slides in histopathology).
C
Case Types & Multipliers
Define each type of work and how many secondary/tertiary units each primary unit generates. Expand a case type to set per-step processing time overrides.
D
Process Steps
Add each step in sequence. Assign it to a swim lane and one or more resource pools. For equipment-constrained steps, assign equipment pools in Section F then link them here.
E
Resource Pools
Define staff groups once. Each pool has funded posts and contracted hours — used to calculate establishment capacity. Multiple pools can be assigned to a single step.
F
Equipment Pools
Define session-based equipment such as dissection benches. Each pool contains one or more benches with a weekly schedule of sessions. Throughput is calculated automatically from your case mix and block times in Section C.
⚠
Dual-capacity model: Enter both establishment capacity (funded, fully-staffed) and available capacity (after absence). The gap between them is your absence cost — often the hidden driver of backlogs.
A
Time Periods
Define the weeks or periods you want to analyse. Enter at least two to reveal demand variation.
B
Demand — Case Type Volumes
Enter the number of primary units (cases, referrals, requests) arriving per case type per period. Volumes at each process step are calculated automatically using the tier multipliers defined in Step 1.
C
External Demand Types Optional
Name discrete external demand streams and route each to the steps it affects. Examples: MDT slide review, referred second opinion, urgent add-ons. Each type contributes flat hours to routed steps without tier expansion.
D
Absence — Per Resource Pool Per Period
Enter hours lost to sickness, annual leave, training, and meetings per pool per period. Available capacity = establishment minus these losses.
